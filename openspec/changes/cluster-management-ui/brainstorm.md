# Cluster Management UI Brainstorm

## Alternatives Considered

### 方案 A：保留内嵌表单（原样）
- **做法**：不改变现有布局，添加集群表单始终显示在页面顶部
- **優點**：无需 UI 改动，实现最简单
- **缺點**：用户每次访问都要看到表单，占用屏幕空间
- **為何未採用**：不符合用户要求的"默认展示集群列表"

### 方案 B：新建按钮 + Modal 弹窗（采用）
- **做法**：表单移入 Modal，页面默认只展示集群列表，点击「新建集群」按钮打开弹窗
- **優點**：列表优先，弹窗交互直观
- **缺點**：多一步点击
- **為何採用**：最简单满足需求，改动最小，符合现有 UI 模式

### 方案 C：侧边栏抽屉
- **做法**：表单放在右侧抽屉，点击「新建集群」滑出
- **優點**：页面不跳转，可同时浏览列表
- **缺點**：改动更大，需要调整布局
- **為何未採用**：方案 B 已足够简单满足需求

---

## Agreed Approach

**采用方案 B**：新建按钮 + Modal 弹窗

- 页面默认展示集群列表（工具栏 + 列表）
- 工具栏右侧放置「新建集群」按钮
- 点击按钮打开 Modal 弹窗，内含 name + kubeconfig 表单
- 提交成功后自动关闭弹窗并刷新列表
- 取消或点击遮罩关闭弹窗

## Key Decisions

- 复用现有 `Modal` 组件，不造新组件
- 表单字段保持 name + kubeconfig 不变
- API 复用 `createCluster`
- Modal 提交后直接关闭，不保持打开（用户明确需求）
