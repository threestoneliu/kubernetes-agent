# Design Summary

## Layout: 三栏恒显 (Three-Column Persistent)

用户从 4 个布局方案中选定 Layout 2：三栏同时可见，左侧图标导航 + 中间会话列表 + 右侧聊天区。三栏布局优点：会话切换无需抽屉滑出，始终可见当前会话上下文，信息密度高。

**具体规格：**
- 左侧导航栏：宽 60px，仅显示图标（💬/☸/🛡），hover 显示文字提示，底部设置图标
- 中间会话面板：宽 240px，顶栏含 logo/新建按钮/搜索框，下方会话列表，active 行高亮
- 右侧聊天区：flex 1，含顶部标题栏（含集群名）/ 消息流 / 底部输入框

## Color Theme: Dark Pro (深色专业)

用户多次强调"不够眼前一亮"、"简约大气"。选定 Dark Pro 配色（深石墨背景 + 霓虹蓝强调），科技感强、层次清晰。

**色值体系：**
- 背景：#0d1117（深石墨）、面板：#161b22、卡片：#161b22
- 文字：#e6edf3（亮白）、次文字：#8b949e
- 强调：#58a6ff（霓虹蓝）、成功：#238636、工具输出：#7ee787
- 边框：#21262d

## Theme Switching: CSS Variable + React Context

用户明确要求"支持主题切换"。实现方式：
- CSS 变量全部定义在 `:root[data-theme="dark"]` 和 `:root[data-theme="light"]` 下
- React Context 提供 `ThemeContext`，`<html data-theme={theme}>` 由顶层组件设置
- 主题偏好存储在 `localStorage`，页面加载时读取
- 首次访问默认深色（Dark Pro），用户可切换为浅色（Light）

## Alternatives Considered

### 方案 A：云栖白（浅色）
- 浅白背景 + 纯黑白灰，用户评价"不够眼前一亮"
- 三栏浅色布局容易显得"朴素"而非"简约大气"
- 未采用

### 方案 B：琥珀暖光
- 暖白背景 + 琥珀色强调，有亲和力但科技感偏弱
- 与 Kubernetes/代码工具氛围略有不搭
- 未采用

### 方案 C：沉浸式单栏 + 抽屉
- 全屏聊天体验好，但会话切换需滑出抽屉，频繁使用不便
- 信息密度低，每次只能看到聊天
- 未采用

## Agreed Approach

**Layout 2（三栏恒显）+ Dark Pro 配色 + 主题切换**

三栏布局确保会话管理始终可见，Dark Pro 配色提供科技感和视觉冲击力，主题切换满足用户个性化需求。

## Key Decisions

1. **三栏布局**：左侧图标导航（60px）+ 中间会话面板（240px）+ 右侧聊天（flex:1）
2. **Dark Pro 深色为主，默认主题**：首次访问使用深色，localStorage 持久化
3. **浅色主题作为第二主题**：在深色基础上反转背景/文字色阶，保持同样的视觉层级
4. **主题切换入口**：置于左侧导航栏底部（⚙图标旁），点击弹出主题选择菜单
5. **CSS 变量隔离**：所有颜色通过 CSS 变量引用，无硬编码色值
6. **布局与主题正交**：布局结构不变，仅 CSS 变量值变化，切换主题不触发重排
