# Brainstorm · Session Management

> Generated by `superpowers:brainstorming` on 2026-06-17.
> Source: openspec/changes/k8s-natural-language-agent/ retrospective §6 promote candidates,
> post-archive `docs/roadmap.md` follow-up #6 (UI 打磨 · 审计日志页/可视化 diff/Plan 历史回放),
> user request "添加会话管理的能力,当前会话没有管理能力,无法选择会话继续聊天,也没有办法管理历史会话".

## Design Summary

在 ChatView 内嵌入 300px 宽的左侧栏 `SessionsPanel`,展示历史会话列表 + 顶部 toolbar(新建/搜索/排序/导出/清空),支持点选切换、重命名 inline、hover 菜单(导出 MD/JSON/删除)、活跃 session 删除禁用、未保存草稿保留。

后端扩展 `GET /api/sessions?q=&sort=&order=&limit=&offset=`、新增 `PUT/DELETE /api/sessions/{id}`、`GET /api/sessions/{id}/export?format=md|json`、`DELETE /api/sessions`(bulk 清空)。Store 层加 SQL `WHERE title LIKE ? ORDER BY ... LIMIT/OFFSET`,`ON DELETE CASCADE` 自动清 messages/plans/audit。

## Alternatives Considered

### 方案 A:服务端搜索/排序/分页(已采用)
- **做法**:`GET /api/sessions` 加 `q`/`sort`/`order`/`limit`/`offset` query 参数,后端 SQL `WHERE title LIKE ? ORDER BY ... LIMIT ? OFFSET ?`
- **优点**:100 条上限放宽后仍能 scale;前端无状态;SQL 即可实现
- **缺点**:改 1 个 handler + 加 query 参数解析 + 1 个 store 方法
- **为何采用**:改动量小,后续要全文搜索可以再叠加 FTS5

### 方案 B:客户端全量过滤
- **做法**:后端不变,前端 `useEffect` fetch 全部 100 条,`useMemo` 过滤排序
- **优点**:0 后端改动,前端实现最简单
- **缺点**:100 条上限后性能差;搜索框跨网络往返无意义;不解决根本的 "100 条不够" 问题
- **为何未采用**:不解决规模问题,且 UI 已经有 100 条 hard cap

### 方案 C:SQLite FTS5 全文索引
- **做法**:建 `sessions_fts` 虚拟表 + 触发器同步 `title` + `messages.text`,`MATCH` 查询
- **优点**:全文搜索快,长会话搜内容能命中
- **缺点**:迁移脚本 + 触发器复杂度高;MVP 不需要,标题搜索够用
- **为何未采用**:over-engineering,后续真有需求再加

## Agreed Approach

**方案 A** — 服务端搜索/排序/分页。理由:
- 当前 sessions 表已 ON DELETE CASCADE,无需 schema 迁移
- 标题 LIKE 搜索 + ORDER BY updated_at DESC + LIMIT 100 是 MVP 合理上限
- UI 端用 2-pane 布局:左 SessionsPanel 300px + 右 ChatView(现有)
- 导出 MD + JSON 两种格式,各一份,直链下载

## Key Decisions

1. **Layout**:ChatView 内部 2-pane flex;SessionsPanel 300px 宽,折叠按钮收起到 0px
2. **Edge case**:
   - 删除前弹 confirm 模态(避免误删)
   - 活跃 streaming session 后端检查 + 前端按钮 disabled
   - 切换会话保留 input 草稿(per-session draft map)
3. **Export**:直链 `<a download>` 触发,后端 `Content-Disposition: attachment; filename=session-<id>.{md,json}`;不需要前端二次处理
4. **Search**:server-side,标题 `LIKE '%q%'`,case-insensitive via `COLLATE NOCASE`
5. **Sort**:支持 `updated_at` / `created_at` / `title` × `asc` / `desc`,默认 `updated_at desc`
6. **Bulk delete**:`DELETE /api/sessions` 需要前端二次确认;返回删除数量
7. **Rename**:inline edit,失焦或 Enter 触发 PUT,Escape 取消
